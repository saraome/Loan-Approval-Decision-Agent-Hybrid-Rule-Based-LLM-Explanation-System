
# System Architecture

## Overview

The **Loan Approval Decision Agent** uses a hybrid architecture that separates:

- Deterministic rule-based decision making
- LLM-based explanation generation

This separation ensures that **financial decisions remain deterministic, auditable, and safe**, while still benefiting from natural language explanations generated by a language model.

---

# High-Level Architecture

User Query
↓
Variable Extraction
↓
Rule Matching Engine
↓
Decision Selection
↓
LLM Explanation Layer
↓
Structured Response

---

# Core Components

## 1. User Query Interface

The system receives a natural language request such as:

"I need a loan of 15000 dollars."

The query may contain:

- loan amount
- credit score

---

## 2. Variable Extraction

The extraction module parses the user query using **regular expressions**.

Extracted variables:

- amount
- credit_score

Example:

Query:
"I want a loan of 25000 and my credit score is 720"

Output:

amount = 25000
credit_score = 720

---

## 3. Rule Matching Engine

The rule engine evaluates extracted variables against predefined rules stored in a **CSV file**.

Example rule:

amount > 20000 AND credit_score < 700

Rules are parsed and evaluated using safe Python logic without using `eval()`.

---

## 4. Decision Selection

### Level 1

Strategy:
First matching rule wins.

### Level 2

Strategy:
Rule priority based on risk level.

Priority order:

High > Medium > Low

If two rules share the same risk level, the **first matching rule wins**.

---

## 5. LLM Explanation Layer

After the decision is determined by the rule engine, the LLM generates an explanation.

The LLM receives:

- user query
- selected rule
- decision
- risk level

Example output:

Decision: MANUAL_REVIEW  
Risk Level: Medium  
Reasoning: Loan amount falls within the medium loan range requiring manual review.

The LLM **cannot modify the decision**.

---

# Safety Design

The system enforces multiple safety guarantees:

1. No LLM decision authority
2. Deterministic rule execution
3. No unsafe `eval()` execution
4. Structured output format

This design ensures:

- reliability
- auditability
- compliance

---

# Hybrid AI Architecture

This project demonstrates a **Hybrid AI Design Pattern**:

Symbolic AI (Rule Engine)
+
Neural AI (LLM Explanation)

Benefits:

- deterministic financial decisions
- explainable outputs
- reduced hallucination risk
